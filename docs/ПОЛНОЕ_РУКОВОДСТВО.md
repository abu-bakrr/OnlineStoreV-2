# О шаблоне

Это универсальный шаблон для создания мобильного и десктопного интернет-магазина. Весь магазин настраивается через JSON файлы - не нужно изменять код!

## Что можно настроить:

- ✅ Название магазина и описание
- ✅ Логотип
- ✅ Цветовую схему (все цвета)
- ✅ Валюту (символ, код, позицию)
- ✅ Контакт менеджера в Telegram
- ✅ Тексты интерфейса
- ✅ Шрифты и их начертание
- ✅ Размер логотипа

## Что находится в базе данных:

- Категории товаров
- Товары (название, описание, цена, фото, цвета, характеристики)
- Пользователи
- Корзины и избранное
- Все токены ботов и API ключи (через .env)

### Параметры подробно:

#### Основные настройки:

- **shopName** - название магазина (отображается в шапке)
- **description** - описание (для SEO)
- **logo** - путь к логотипу (обычно `/config/logo.svg`)

#### Валюта:

- **symbol** - символ валюты (`₽`, `$`, `€`, `₸`, и т.д.)
- **code** - код валюты (`RUB`, `USD`, `EUR`, `KZT`)
- **position** - позиция символа: `"after"` (1000₽) или `"before"` ($1000)

#### Контакт:

- **managerContact** - ваш Telegram для связи (например: `@username`)

#### Цветовая схема:

Все цвета в формате HEX (#RRGGBB):

- **background** - фон страницы
- **foreground** - основной цвет текста
- **primary** - основной акцентный цвет
- **card** - фон карточек
- **border** - цвет границ
- И т.д.

**Примеры цветовых схем:**

Пастельный розовый (по умолчанию):

```json
"primary": "#EADCF0",
"primaryForeground": "#1A1A1A"
```

Голубой минимализм:

```json
"primary": "#E3F2FD",
"primaryForeground": "#1A1A1A"
```

Тёмный модерн:

```json
"primary": "#2C3E50",
"primaryForeground": "#FFFFFF"
```

#### Опции сортировки:

Можете изменить текст и эмодзи:

```json
{ "id": "new", "label": "Новое", "emoji": "🆕" }
```

#### UI настройки:

- **maxWidth** - максимальная ширина интерфейса (`420px` для телефонов)
- **productsPerPage** - товаров на странице
- **showCategoryIcons** - показывать ли иконки категорий
- **showPriceFilter** - показывать ли фильтр по цене

#### Тексты:

Все надписи кнопок можно изменить

#### Шрифты:

- **fontFamily** - название шрифта (`Inter`, `Roboto`, `Open Sans`)
- **fontFile** - путь к файлу шрифта (если свой шрифт)
- **productName/price/description** - толщина шрифта (400-700)

#### Размер логотипа:

- **logoSize** - размер в пикселях (рекомендуется 32-48)

## Шаг 3: Добавление товаров и категорий

Товары и категории хранятся в базе данных PostgreSQL.

### На VPS после установки:

#### Вариант A: Через SQL

```bash
# Подключитесь к VPS
ssh root@ВАШ_IP

# Подключитесь к базе данных
sudo -u postgres psql shop_db

# Добавьте категорию
INSERT INTO categories (id, name, icon)
VALUES (gen_random_uuid(), 'Электроника', '💻');

# Посмотрите ID категории
SELECT id, name FROM categories;

# Добавьте товар (замените CATEGORY_ID на ID из предыдущей команды)
INSERT INTO products (id, name, description, price, images, category_id)
VALUES (
  gen_random_uuid(),
  'iPhone 15 Pro',
  'Флагманский смартфон Apple',
  89990,
  ARRAY['https://example.com/iphone.jpg'],
  'CATEGORY_ID'
);

# Выход
\q
```

#### Вариант Б: Через Python скрипт

Создайте файл `add_products.py`:

```python
import psycopg2
import os

# Подключение к БД
conn = psycopg2.connect(os.environ['DATABASE_URL'])
cur = conn.cursor()

# Добавить категорию
cur.execute("""
    INSERT INTO categories (name, icon)
    VALUES (%s, %s)
    RETURNING id
""", ('Электроника', '💻'))
category_id = cur.fetchone()[0]

# Добавить товары
products = [
    ('iPhone 15 Pro', 'Флагманский смартфон', 89990, ['https://example.com/iphone.jpg']),
    ('MacBook Pro', 'Мощный ноутбук', 199990, ['https://example.com/macbook.jpg']),
]

for name, desc, price, images in products:
    cur.execute("""
        INSERT INTO products (name, description, price, images, category_id)
        VALUES (%s, %s, %s, %s, %s)
    """, (name, desc, price, images, category_id))

conn.commit()
cur.close()
conn.close()

print("Товары добавлены!")
```

Запустите:

```bash
cd /home/shopapp/app
source venv/bin/activate
python3 add_products.py
```

```bash
# На VPS
```

## Приватный GitHub репозиторий

Если ваш репозиторий приватный, нужно настроить доступ:

### Вариант A: SSH ключ (рекомендуется)

```bash
# На VPS создайте SSH ключ
ssh-keygen -t ed25519 -C "vps-deploy"

# Нажмите Enter 3 раза (без пароля)

# Покажите публичный ключ
cat ~/.ssh/id_ed25519.pub
```

**Скопируйте весь вывод** и добавьте в GitHub:

1. Откройте GitHub → Settings
2. SSH and GPG keys
3. New SSH key
4. Вставьте ключ
5. Save

Теперь используйте SSH URL:

```bash
GITHUB_REPO="git@github.com:ВАШ_USERNAME/ВАШ_РЕПОЗИТОРИЙ.git" \
curl -fsSL https://... | sudo bash
```

### Вариант Б: Personal Access Token

1. GitHub → Settings → Developer settings
2. Personal access tokens → Generate new token
3. Выберите права: `repo` (full control)
4. Generate token
5. **Скопируйте токен** (больше не увидите!)

Используйте URL с токеном:

```bash
GITHUB_REPO="https://ВАШ_ТОКЕН@github.com/ВАШ_USERNAME/ВАШ_РЕПОЗИТОРИЙ.git" \
curl -fsSL https://... | sudo bash
```

# Настройка домена и SSL

## Подключение своего домена

### Шаг 1: Настройте DNS

В панели управления вашего домена (Cloudflare, REG.RU, и т.д.):

1. Создайте A-запись:
   - **Тип**: A
   - **Имя**: @ (или ваш поддомен)
   - **Значение**: IP_АДРЕС_VPS
   - **TTL**: 300 (или Auto)

2. Подождите 5-60 минут (пока DNS обновится)

3. Проверьте командой:
   ```bash
   ping ваш-домен.com
   ```
   Должен показать IP вашего VPS

### Шаг 2: Настройте Nginx на VPS

```bash
# Подключитесь к VPS
ssh root@ВАШ_IP

# Запустите скрипт настройки домена
cd /home/shopapp/app
sudo ./setup_domain.sh
```

Скрипт спросит:

```
Введите ваш домен: ваш-домен.com
```

Готово! Теперь сайт доступен по домену.

## Установка SSL сертификата (HTTPS)

### Шаг 1: Убедитесь что домен настроен

```bash
ping ваш-домен.com
```

Должен показать IP вашего VPS.

### Шаг 2: Установите SSL

```bash
# На VPS
cd /home/shopapp/app
sudo ./setup_ssl.sh
```

Скрипт спросит:

```
Введите ваш домен: ваш-домен.com
Введите email для уведомлений: ваш@email.com
```

### Что происходит:

1. Устанавливается Certbot
2. Получается бесплатный SSL от Let's Encrypt
3. Настраивается автоматическое продление
4. Nginx перенастраивается на HTTPS

**Готово!** Теперь сайт доступен по `https://ваш-домен.com`

### Автоматическое продление

SSL сертификат действителен 90 дней и продлевается автоматически.

Проверить можно:

```bash
sudo certbot renew --dry-run
```

# Админ-панель

Админ-панель доступна по адресу: `http://ваш-сайт/admin`

## Первый вход в админку

### Способ 1: Автоматический (рекомендуется)

1. Зарегистрируйте аккаунт на сайте (обычная регистрация)
2. Перейдите на `/admin/login`
3. Введите email и пароль вашего аккаунта
4. Если в системе нет администраторов — вы автоматически станете первым админом (superadmin)

### Способ 2: Через базу данных

```bash
ssh root@ВАШ_IP
sudo -u postgres psql shop_db

-- Сделайте пользователя администратором
UPDATE users SET is_admin = TRUE, is_superadmin = TRUE WHERE email = 'ваш@email.com';

\q
```

## Возможности админ-панели

### Вкладка "Товары"

- **Просмотр** всех товаров с поиском и фильтрацией
- **Добавление** нового товара с загрузкой фото через Cloudinary
- **Редактирование** — изменение названия, описания, цены, категории
- **Удаление** товаров

При добавлении товара можно указать:

- Название и описание
- Цену
- Категорию
- До 5 изображений (загружаются через Cloudinary)
- Цвета (например: Красный, Синий, Зеленый)
- До 2 дополнительных атрибутов (например: Размер с вариантами S, M, L, XL)

### Вкладка "Категории"

- **Создание** новых категорий с иконкой (эмодзи)
- **Редактирование** названия и иконки
- **Удаление** категорий
- **Порядок сортировки** — указывается числом

### Вкладка "Заказы"

- **Просмотр** всех заказов с информацией о покупателе
- **Фильтрация** по статусу (ожидает, оплачен, отправлен, доставлен)

# Платежные системы

Система поддерживает несколько способов оплаты. Методы отображаются на сайте только если они настроены.

## Настройка через админ-панель

1. Войдите в `/admin`
2. Перейдите в "Настройки"
3. Настройте нужные платежные системы

## Перевод на карту

Самый простой способ приёма платежей.

### Настройка в `config/settings.json`:

```json
{
	"payment": {
		"cardTransfer": {
			"enabled": true,
			"cardNumber": "8600 1234 5678 9012",
			"cardHolder": "Иванов Иван Иванович",
			"bankName": "Uzcard"
		}
	}
}
```

### Как это работает:

1. Покупатель оформляет заказ
2. Видит реквизиты карты для перевода
3. Делает перевод через приложение банка
4. Загружает скриншот чека
5. Администратор видит чек в разделе "Заказы" и подтверждает оплату

## Click

Онлайн оплата для Узбекистана.

### Настройка:

1. Зарегистрируйтесь в [Click Merchant](https://merchant.click.uz)
2. Получите `Merchant ID` и `Service ID`
3. Создайте секретный ключ в кабинете

### В `config/settings.json`:

```json
{
	"payment": {
		"click": {
			"enabled": true,
			"merchantId": "12345",
			"serviceId": "67890"
		}
	}
}
```

### Переменные окружения (секретные):

```bash
CLICK_MERCHANT_ID=12345
CLICK_SERVICE_ID=67890
CLICK_SECRET_KEY=ваш_секретный_ключ
```

### Webhook URLs (укажите в кабинете Click):

```
Prepare: https://ваш-домен.com/api/webhooks/click/prepare
Complete: https://ваш-домен.com/api/webhooks/click/complete
```

## Payme

Онлайн оплата для Узбекистана.

### Настройка:

1. Зарегистрируйтесь в [Payme Business](https://business.payme.uz)
2. Получите `Merchant ID`
3. Создайте ключ для проверки запросов

### В `config/settings.json`:

```json
{
	"payment": {
		"payme": {
			"enabled": true,
			"merchantId": "your_merchant_id"
		}
	}
}
```

### Переменные окружения:

```bash
PAYME_MERCHANT_ID=your_merchant_id
PAYME_KEY=your_secret_key
```

### Webhook URL:

```
https://ваш-домен.com/api/webhooks/payme
```

## Uzum Bank

Онлайн оплата для Узбекистана.

### Настройка:

1. Подключитесь к Uzum Business
2. Получите `Merchant ID` и `Service ID`
3. Получите секретный ключ для HMAC подписи

### В `config/settings.json`:

```json
{
	"payment": {
		"uzum": {
			"enabled": true,
			"merchantId": "your_merchant_id",
			"serviceId": "your_service_id"
		}
	}
}
```

### Переменные окружения:

```bash
UZUM_MERCHANT_ID=your_merchant_id
UZUM_SERVICE_ID=your_service_id
UZUM_SECRET_KEY=your_secret_key
```

### Webhook URLs:

```
Check: https://ваш-домен.com/api/webhooks/uzum/check
Create: https://ваш-домен.com/api/webhooks/uzum/create
Confirm: https://ваш-домен.com/api/webhooks/uzum/confirm
Reverse: https://ваш-домен.com/api/webhooks/uzum/reverse
```

## Безопасность платежей

- **Секретные ключи** хранятся только в переменных окружения, никогда в коде или конфигах
- **Все webhook'и** проверяют подпись/авторизацию перед обработкой
- **Суммы заказов** вычисляются на сервере из цен в БД (не доверяем клиенту)
- **Данные шифруются** при хранении в БД с помощью Fernet

# Telegram уведомления

Система отправляет уведомления о новых заказах администратору в Telegram.

3. Введите имя бота (например: "My Shop Notifications")
4. Введите username бота (например: `myshop_notify_bot`)
5. Скопируйте токен бота

### Шаг 2: Узнайте ваш Chat ID

1. Напишите боту [@userinfobot](https://t.me/userinfobot) команду `/start`
2. Скопируйте ваш ID (число)

### Шаг 3: Настройте переменные окружения

```bash
# Добавьте в .env файл на VPS
TELEGRAM_BOT_TOKEN=123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TELEGRAM_ADMIN_CHAT_ID=987654321
```

### Шаг 4: Включите уведомления

В `config/settings.json`:

```json
{
	"telegramNotifications": {
		"enabled": true,
		"adminChatId": "987654321"
	}
}
```

Или через админ-панель: Настройки → Telegram уведомления.

# Email и сброс пароля

Система поддерживает сброс пароля через email.

## Как это работает

1. Пользователь нажимает "Забыли пароль?" на странице входа
2. Вводит свой email
3. Получает письмо со ссылкой для сброса
4. Переходит по ссылке и вводит новый пароль
5. Ссылка действительна 1 час и работает только один раз

## Настройка SMTP

### Через переменные окружения:

```bash
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your@gmail.com
SMTP_PASSWORD=your_app_password
SMTP_FROM_EMAIL=your@gmail.com
SMTP_FROM_NAME=Мой Магазин
SMTP_USE_TLS=true
```

### Через админ-панель:

Настройки → SMTP → Заполните все поля

### Настройка для Gmail:

1. Включите двухфакторную аутентификацию в Google аккаунте
2. Создайте пароль приложения:
   - Google Account → Security → App passwords
   - Создайте пароль для "Mail"
3. Используйте этот пароль в `SMTP_PASSWORD`

### Настройка для Yandex:

```bash
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USE_TLS=false  # используется SSL
```

### Настройка для Mail.ru:

```bash
SMTP_HOST=smtp.mail.ru
SMTP_PORT=465
SMTP_USE_TLS=false  # используется SSL
```

## Проверка работы

После настройки:

1. Перейдите на `/forgot-password`
2. Введите email существующего пользователя
3. Проверьте почту (включая папку "Спам")

# Решение проблем

## Белая страница / Ошибки 403 Forbidden

### Проблема:

Сайт не загружается, в браузере белая страница.

### Решение:

```bash
# Исправьте права доступа
cd /home/shopapp/app
sudo ./fix_permissions.sh

# Перезапустите Nginx
systemctl restart nginx
```

Откройте сайт снова - должно заработать.

## Приложение не запускается

### Проверьте логи:

```bash
# Логи приложения
journalctl -u shop-app -n 100

# Статус
systemctl status shop-app
```

### Типичные ошибки:

**1. Ошибка подключения к БД:**

```bash
# Проверьте PostgreSQL
systemctl status postgresql

# Проверьте пароль в .env
cat /home/shopapp/app/.env
```

**2. Порт занят:**

```bash
# Проверьте какой процесс использует порт 5000
sudo lsof -i :5000

# Убейте процесс (замените PID)
kill -9 PID
```

**3. Не собран фронтенд:**

```bash
cd /home/shopapp/app
sudo -u shopapp npm run build
systemctl restart shop-app
```

## Git pull не работает

### Проблема:

`git pull` выдаёт ошибки или не обновляет код.

### Решение:

```bash
# Сбросить локальные изменения
cd /home/shopapp/app
sudo -u shopapp git stash

# Обновить
sudo -u shopapp git pull

# Или принудительно
sudo -u shopapp git fetch origin
sudo -u shopapp git reset --hard origin/main
```

## SSL сертификат не устанавливается

### Проблема:

Certbot выдаёт ошибку при установке SSL.

### Решение:

**1. Проверьте DNS:**

```bash
ping ваш-домен.com
```

Должен показывать IP вашего VPS.

**2. Проверьте порты 80 и 443:**

```bash
sudo ufw status
# Должно быть разрешено: 80/tcp, 443/tcp
```

**3. Попробуйте ещё раз:**

```bash
cd /home/shopapp/app
sudo ./setup_ssl.sh
```

## База данных не подключается

### Проверка:

```bash
# PostgreSQL запущен?
systemctl status postgresql

# Можно подключиться?
sudo -u postgres psql -c "SELECT 1"

# Проверьте DATABASE_URL
cat /home/shopapp/app/.env
```

### Решение:

```bash
# Перезапустите PostgreSQL
systemctl restart postgresql

# Проверьте пользователя БД
sudo -u postgres psql -c "\du"

# Пересоздайте пользователя если нужно
sudo -u postgres psql <<EOF
DROP USER IF EXISTS shop_user;
CREATE USER shop_user WITH PASSWORD 'ваш_пароль';
GRANT ALL PRIVILEGES ON DATABASE shop_db TO shop_user;
EOF
```

```bash
sudo crontab -e
```
